Overview

An operating system you can read end to end

NyxOS boots via Multiboot2 (GRUB-compatible), enters long mode with 4-level paging, and provides a clean foundation for kernel development: a preemptive scheduler, ring-3 userspace with ELF64 and syscalls, a windowed GUI, and a from-scratch network stack — all built up one honest, verified increment at a time.

Features

What's inside the kernel

Every subsystem is written from scratch and verified in QEMU.

⏻

Boot & initialization

Multiboot2, x86_64 long mode, GDT/IDT with a full exception set, PIT at 1000 Hz, Local + I/O APIC, and PS/2 keyboard & mouse drivers.

▤

Memory management

Bitmap physical allocator, a 16 MB kernel heap, 4-level paging with a higher-half mapping, per-process page tables, and NX + SMEP hardening.

⚙

Processes & scheduler

Preemptive weighted round-robin over kernel threads and ring-3 processes, with full shell job control — exec, spawn, jobs, wait, kill, nice.

◈

ELF userspace

An ELF64 loader, an embedded initramfs, fork() with copy-on-write, 44 syscalls via syscall/sysret, a minimal libc, isolated per-process file descriptors, and the 🌙 Nyx C language runtime for first-class userspace programming.

❯

Shell

40+ built-in commands with Tab completion, environment-variable expansion, command history, I/O redirection, pipelines, job control (exec/spawn/jobs/wait/kill), and nice/renice priority control.

🖧

Networking

RTL8139 driver, ARP/IPv4/UDP/ICMP/DHCP/DNS, and a full TCP stack: retransmission with RTO, passive open (listen/accept), loopback, an HTTP client & server, nc full-duplex, and poll() I/O multiplexing.

🗔

GUI compositor

Up to 32 windows with z-ordering, drag & resize, four workspaces, a taskbar and Start menu — plus a Terminal, File Manager, Editor, Paint and more.

🖴

Filesystem

A ramdisk VFS with an EXT2 read/write driver auto-mounted at /mnt, per-fd byte offsets, and persistence across reboots on a disk image.

♪

Multimedia

A Bochs VBE framebuffer (1024×768×32), a Sound Blaster 16 driver (DMA + IRQ + mixer), PC-speaker tones, and a DOOM port with sound.

Architecture

How it fits together

A higher-half kernel, isolated user address spaces, and an interrupt-driven core.

Boot flow

  1. GRUB / Multiboot2 hands off with a memory map + framebuffer.
  2. Long mode — GDT, 64-bit code/data, TSS, IDT with IST stacks.
  3. Paging — identity-map low memory, mirror the kernel into the higher half.
  4. Drivers — PIT, APIC, PS/2, RTL8139, ATA/EXT2, VBE, SB16.
  5. Login — PBKDF2-hashed credentials (nyx / nyx).
  6. Desktop — the window compositor launches as a scheduled process.

Memory layout

0xFFFFFF80_00000000Higher-half kernel (PML4[511])
0x00000000_00100000Kernel linked low (1 MB)
user halfPer-process, isolated (no identity map)
NX + SMEPUser pages non-exec; kernel can't run user code

Interrupt core

The PIT (ISA IRQ0) is routed via the I/O APIC on pin 2 → vector 32, driving a 1000 Hz tick. The scheduler preempts on that tick; syscalls and IRQs switch CR3 on entry so each process sees only its own address space.

Manual

Getting started

Build the kernel, boot it in QEMU, and log in.

1

Clone & build

bash · Linux / WSL
# cross-compiler (x86_64-elf-gcc) or host gcc -m64
git clone https://github.com/kazah-png/nyx-os.git
cd nyx-os
make -C kernel        # → kernel/nyx-kernel.bin
PowerShell · Windows
.\build.ps1           # WSL cross-compile → kernel + bootable ISO
2

Run in QEMU

bash
# GUI desktop + networking + disk
qemu-system-x86_64 -cdrom NyxOS.iso -m 256M -no-reboot \
  -hda ext2-test.img -nic user,model=rtl8139 -display sdl

Serve HTTP from the guest by forwarding a host port: add hostfwd=tcp::8080-:80 to the NIC, run tcpserve 80, then curl localhost:8080.

3

Log in

The boot animation leads to a framebuffer login screen. The default account is nyx / nyx (PBKDF2-HMAC-SHA256). On success, the NyxOS Desktop launches; with no disk, it falls back to a text shell.

Networking

A real TCP/IP stack

From the NIC driver up to an HTTP server — every layer written and debugged from the wire.

HTTP · DNS · ncclient, server, full-duplex poll()

RTL8139PCI NIC · TX/RX rings

Highlights

  • DHCP client — DISCOVER → OFFER → REQUEST → ACK, auto-configures IP, netmask & gateway.
  • ping with statistics — real round-trip timing, per-packet sequencing, and a min/avg/max + loss summary. NyxOS also answers echo requests.
  • TCP retransmission — each connection buffers its outstanding segment and resends it on an RTO with exponential backoff; a lost SYN or data segment recovers automatically.
  • Passive open — listen/accept spawn per-client connections. tcpserve answers a real curl from the host with HTTP/1.1 200 OK.
  • Userspace sockets — socket/connect/sendto/recvfrom/bind/listen/accept for both TCP and UDP from ring 3.
  • poll() I/O multiplexing — SYS_POLL (syscall 43) waits on sockets, pipes, and stdin with timeout. Powers full-duplex nc.
  • nc (netcat) — full-duplex TCP/UDP client and TCP server, bridging stdin/stdout via poll().
  • Loopback — a self-contained 127.0.0.1 path lets the whole stack be tested in-guest (tcploop, ping 127.0.0.1).
Releases

The road so far

Selected milestones from a from-scratch kernel to a networked desktop OS.

v1.x

Foundations

Base kernel, ramdisk VFS + shell commands, and the first real networking (RTL8139, ARP/IP/UDP/ICMP).

v2.x

Multitasking, GUI & disks

Preemptive multitasking, a full TCP stack, the window compositor, PC-speaker audio, EXT2 read support, and a Sound Blaster 16 driver.

v3.x

Userspace

ELF loader, initramfs, per-process paging, ring-3 execution, syscalls, an RTC driver and a minimal libc.

v4.x

64-bit & hardening

Full x86_64 long mode, higher-half kernel, user/kernel page-table isolation, and NX + SMEP with APIC init.

v5.0–5.6

Apps, login & ring-3

A full GUI app suite, a boot animation + login screen, security hardening of the syscall boundary, and ring-3 userspace that actually runs.

v5.7.x

Scheduler & the network arc

Preemptive scheduling with job control & nice, per-process fds, drag-and-drop in the File Manager, and a networking sprint: loopback + ICMP, TCP retransmission, multi-segment receive, passive open, and an HTTP server that answers real clients.

v5.8.0

🌙 Nyx C language runtime

Nyx C — a Go/Zig-inspired typed subset of C that transpiles to C and links against the nyxrt runtime. The first .nyx program prints "hola desde nyx c! pid=5" via string interpolation and direct syscalls.

v5.8.1

waitpid(): fork is complete

SYS_WAITPID (11) — a ring-3 parent reaps its children and collects their exit codes.

v5.8.3

Anonymous pipes & blocking IPC

pipe() (SYS_PIPE, 12) with kernel ring buffers, reference-counted ends, and blocking read() — real cross-process IPC. Per-process syscall stacks enable true blocking in ring 0.

v5.8.3–5.8.25

Userspace & terminal polish

execve argv, dup2, getdents, coreutils (cat/wc/ls), shell background jobs, interactive REPL, lazy sbrk, signals, mmap/mprotect, /dev, /proc, shell line editing & history, Tab completion, ANSI SGR color — a complete ring-3 environment.

v5.8.26

ANSI colors & new nyxfetch

The GUI terminal renders ESC[…m SGR colors (fg, bg, bold, reverse) with a per-character color scrollback. nyxfetch gets a neofetch-style ASCII fox on the left, facts on the right — by @kurawi-debug.

v5.8.27–5.8.40

Desktop, visual refresh & double buffering

Unified crescent-moon logo across login, nyxfetch and panic screen. Wallpaper app with 11-color palette picker. Double-buffered compositor (no flicker). Shell operators &&/||/;, globbing (*/?), ~ expansion, and job control (Ctrl-Z, fg/bg).

v5.8.41–5.8.49

Stability, signals & catchable faults

Graphical kernel panic stop screen. Ring-3 fault isolation (a crash kills the process, not the kernel). less/more pager. Catchable signals (SIGSEGV/SIGFPE/SIGILL) with setjmp/longjmp recovery, sigsetjmp/siglongjmp for multi-fault recovery, and alarm() + SIGALRM.

v5.8.50–5.8.55

Network stack maturity & UDP sockets

Network byte-order consolidation. Userspace TCP sockets with socket/connect/read/write. Server-side bind/listen/accept. Scheduler CR3 fix for mid-syscall preemption. Userspace UDP sockets (sendto/recvfrom). 41 syscalls.

v5.8.56–5.8.70

Userspace sockets, login polish & persistent accounts

nc (netcat), catchable-signal multi-recovery (sigsetjmp/siglongjmp), alarm() + SIGALRM, persistent EXT2 verified end-to-end, poll() I/O multiplexing, full-duplex nc. GUI File Manager browses & opens /mnt files in the Text Editor. Persistent user accounts with random salts, avatar picker & lockout. Taskbar user menu (change picture, logout, per-user home dirs).

v5.8.71

Starter home directory content

New home directories get Documents/, Downloads/ and welcome.txt — only on first creation, so subsequent logins never overwrite user files.

v5.8.72

Minesweeper — GUI game

Classic 9×9 / 10-mine Minesweeper as a compositor window. Lazy mine placement (first click always safe), flagging, chording, face button, timer. Also fixes right-click dispatching to respect z-order instead of slot order.

v5.8.73

wget — userspace HTTP client

Self-contained HTTP GET over TCP sockets with its own DNS-over-UDP resolver. Supports -O output file, -r/-p resolver, and status output. No new syscalls.

v5.8.74

exec forwards argv — tools run with args from the kernel shell

exec <file> a b c and spawn build the child's SysV argv frame and launch it directly. wget, grep, cat all work without /sh.elf. No new syscalls.

v5.8.75

Kernel shell auto-execs userspace ELFs by bare name

When a typed command isn't a builtin, execute_command resolves it to a userspace ELF and runs it with argv. Type whoami or wget directly — no exec prefix needed.

v5.8.76

Tab-completion includes userspace ELFs

Completes not just builtins but also .elf base names from initramfs, with builtins taking priority. Type whoa+Tab → whoami.

v5.8.77

wget follows redirects + open command

wget follows HTTP 3xx redirects (up to 5 hops) and a new open builtin opens files in the GUI Text Editor from the kernel shell.

v5.8.78

Concurrent multi-process sockets hardening

Net-stack re-entrancy serialization, atomic ephemeral-port pick, connection table capacity bumped. New netstorm repro forks 4 processes hammering loopback echo.

v5.8.79

Root-caused the netstorm Heisenbug — Makefile stale-build gap

The v5.8.78 "ultra-fragile timing" bug was a conn-table capacity limit masked by a Makefile that didn't recompile on header changes. GCC auto-dependencies (-MMD -MP) fix it.

v5.8.80

Four POSIX file syscalls — stat/fstat/lseek/getppid

New syscalls 44-47: stat/fstat (size+mode), lseek (SEEK_SET/CUR/END), getppid. Makefile fix: .DEFAULT_GOAL prevents stale builds.

v5.8.81

VFS hardening — MAX_CHILDREN heap OOB fixed + caps raised

Unified vfs_append_child choke point (8 creation paths were missing the bound check). Caps: MAX_INODES 128→256, MAX_CHILDREN 64→128. Initramfs logs warnings instead of silently dropping files.

v5.8.82

dup + rename syscalls + spawned tools inherit terminal cwd

New SYS_DUP(48)/SYS_RENAME(49). create_user_process seeds cwd from the spawning terminal. fsx.elf demo.

v5.8.83

THE pipeline-corruption Heisenbug FIXED — ROM hole + .lbss

Root-caused: init_memory left the firmware hole 0xA0000–0xFFFFF marked FREE, so under memory pressure alloc_page handed out VGA/BIOS ROM frames. Also .lbss/.ldata/.lrodata were unreserved in linker.ld. Issue #20 closed.

v5.8.84

Physical allocator honors multiboot2 memory map

init_memory parses the multiboot2 type-6 tag and frees ONLY type-1 (available RAM) pages — every firmware hole is reserved automatically. Falls back to v5.8.83 behavior when no tag present.

v5.8.85

Demand-grown user stacks with guard page

User stacks are no longer fixed 64 KB — they start at 16 KB and grow on demand up to 512 KB, with a permanent unmapped guard page for clear overflow diagnostics.

v5.9.0-LTS

LTS release — P0.1 closed: preemptible CR3-switch → iretq window fixed

The one release-blocking bug root-caused and fixed with a single cli — closing the preemptible window between user-CR3 reload and iretq. Also: 3 GUI highs (paint divide-by-zero, menu hit-test skew, fb backbuffer repoint, level-triggered menus), VFS fd refcounts, ext2 write-path overhaul verified by e2fsck (9 defects), editor truncation guard, audit/hardening (ELF bounds, user_v2p, FMASK, cld, TSS struct, 22-pixel GUI offset, framebuffer clipping, dynamic linker, fork isolation with mprotect), SMP stages (per-CPU TSS, user procs on APs, spinlocks, TLB shootdown IPIs), and thread/fd table coherence.

v5.9.1

"Nightfall" desktop — central GUI palette & brand purple

kernel/theme.h — every colour is now a named ROLE (THEME_ACCENT, THEME_TEXT, …) instead of scattered fb_rgb() literals. The accent is the brand purple {130,90,210} everywhere — Start menu, taskbar, focused window. Desktop wallpaper rewritten as a nightfall sky with a glowing moon, craters, and a seeded star field. Workspace indicator fixed (light-vs-dark over the wallpaper). All README badges updated to the brand purple.

v5.9.2

Palette reaches the apps — focused window gets the brand colour

The v5.9.1 theme.h roles rolled out to all ten app windows: 206 → 169 fb_rgb() literals converted, 61 named role uses. Window chrome: THEME_TITLE_ACTIVE = brand purple for the focused title bar, THEME_TITLE_INACTIVE = dark grey for the rest. Selection highlights (THEME_SELECTION), panel headers (THEME_PANEL_HEADER), and buttons (THEME_BUTTON) all themed. App content colours kept as data (paint swatches, Minesweeper cells, ANSI palette).

v5.9.4

Desktop icons wrap on narrow resolutions

The icon layout was a single fixed row that ran off the screen at 640×480 (the resolution Settings offers). Now icons are laid out as a grid that re-flows on the framebuffer width. Settings' resolution change handler calls init_desktop_icons() so switching to a smaller mode re-flows them cleanly. At 1024×768 the layout is pixel-identical (13 columns, all nine icons in one row).

v5.9.3

Window frame joins the focus story

The window frame now carries focus too: a focused window gets the brand accent on its top+left edge and a darker accent on bottom+right, so the border and title bar merge into one continuous coloured edge. Unfocused windows keep the neutral grey bevel for depth between stacked windows. Title text and button glyphs unified onto THEME_TITLE_TEXT. Traffic-light buttons untouched (close red, minimise grey, maximise green — control affordance, not branding).
